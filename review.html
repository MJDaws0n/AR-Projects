HGRU AR Quiz App Review

Background

These are the basic specifications of the app:

  • Select Framework: For this project I chose to use Aframe as my small amount of research suggested that it would be easy to use. I also already knew HTML, CSS and JS so it would be simple to adapt my knowledge.
  • Create project structure: As this is a closed source small scale app, only a simple file structures is needed. A folder for assets, markers, images and models.
  • Development space: I originaly used github codespace however as there are only limited hours available that get used quicly, I swiched to just pushing code to a github branch and using a local version of PHP to host the file and Microsft devtunnel for accessing on my phone.

Steps

These are the steps I used to build the app:

  • Design: Beside the (useless) required design, I also made sure to make a genuinly useful design and created every page in figma before making it.
  • Creating pages: All the pages started with blank html pages with the name of them, followed by .html, for example, index.html was the home page and help.html was the help page.
  • Page structure and re-usable assets: Most pages used the default app.css stylesheet I built, this allowed re-usable, easly modifiable, and consistant looking pages.
  • Scanning: Scannng simply uses Aframe with some HTML to check when a maker .patt file is present, then overlaying a 3D model. A border wa harder to add as Aframe is limited so an overlayed image was required to be used (a bit of a "hacky" method).
  • Quiz: The quiz worked by having a blank element for question number, subject, current question, and answer a through to c then reading from quiz.json. It searches through the file, and takes query from the URL then selects the questions associated to it (dynamically updating the total questions based on how many are there), and the answer specified. The questions are then sorted into a random order and the answers placed in a random order.
