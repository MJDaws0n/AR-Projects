HGRU AR Quiz App Evaluation

Overview

The quiz app includes all requested features with many addition features.

Required Functionality

The app includes the following required features:

  • Select start on the opening screen: The app presents the user with a start screen that allows the user to click start. There is a watermark that says By Max that is possition perfectly so it can be replaced with a copyright subtext or footer.
  • Select a topic that will open at question 1: After scanning an AR marker or manually starting the quiz, the quiz starts, instead of starting on question one however, the quiz is started on a random question, this adds diverstity and makes the quiz feel different when it's replayed.
  • Use interaction/layers to choose an answer and be informed whether it is right or wrong: Answer boxes appear in a random order and will be different every time you play the quiz. The quiz questions can be added and modifed in quiz.json file, this allows for expandability. More than 3 questions can also be added to make the quiz more immersive.
  • Select a different answer if the wrong answer has been chosen : This feature is not directly usable while combining the required scoring system as changing an answer would mean the user would always get 100% and not learn anything. So to meet this requirment, after a quiz the user is presented the option to replay the quiz, this allows them to "select a different answer if the wrong answer has been chosen" without compramising the scoring system and also ensure they actually know it as replaying the quiz re-orders the questions and anwsers.
  • Use interaction/layers to move to the next question once the correct answer has been chosen: The app will automatically move you onto the next question 3 seconds after getting a question incorrect, the interaction for this is answering a question, this keeps a fast paced action pack feel the app, it increase retention as you are not requried to click to continue/./
  • Use interaction/layers to move back to the opening screen after the third question: After the final question which - will be the "third question" for the provided assets in the quiz.json file - the app returns to a new screen asking the user if they wish to replay the quiz (accomidating for other requirements) with another option asking if they would like to return to the scan page.

Additional Functionality

The app includes the following additional features:

  • Help page: The app has a help page that explains why scanning the AR marker may not work ans gives some suggestions on fixing it. It also allows manually starting the quiz by clicking a button.
  • Multiple quizzes: Although only one quiz was required all the quizzes are avaiable.
  • Random quiz order: Quizzes can appear in random orders and so can the answer.
  • Error page: If there is a quiz error related to the quiz not being able to be found, or if someone is trying to start a quiz that is not registerd then it presents an error saying Sorry, the current quiz is no longer avaiable.
  • Score page: Once you finish the quiz it will show the score you got. With options restart.

Performance

The application performs well in all scenarios:

  • Speed: Minimal files used, and minimal elements rendered in the browser to improve speed.
  • Efficiency: Hardly any storage is required, the only thing required to run is a browser supporting HTML 5.
  • Reliability: Out of all the testing, there are no found bugs.
